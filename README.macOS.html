EggLedger installation/upgrade instructions

When you first launch EggLedger, macOS will block it because the app is not signed with a recognized developer certificate. Follow these steps to allow it to run:

  1. Double-click EggLedger.app. macOS will show a dialog saying the app “cannot be opened because Apple cannot verify it.” Click Done to dismiss the dialog — do not try to open it from this dialog.
  2. Open System Settings → Privacy & Security. Scroll down to the Security section. You should see a message about EggLedger being blocked, with an Open Anyway button next to it. Click Open Anyway.

    The Open Anyway button disappears after approximately one hour. If you do not see it, try launching EggLedger.app again first to re-trigger the block, then return to Privacy & Security.

  3. macOS will show a follow-up security confirmation dialog. Click Open to confirm.

EggLedger will now launch normally. You only need to do this once per version.

If you’re upgrading to a new version

Drag EggLedger.app to where the old version is and overwrite the old app bundle. This ensures the new version finds your existing data without redownloading everything. You will need to repeat the Privacy & Security steps above for each new version.

You can move the entire folder containing EggLedger.app to anywhere you like. Do NOT move the app separately to a shared folder like /Applications; the app writes data to the folder where EggLedger.app resides, and you don’t want to pollute a shared folder.
